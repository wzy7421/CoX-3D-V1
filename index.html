Explainable Generative 3D Design
driven by Social Media Semantics

CoX‑3D is an end‑to‑end pipeline connecting user‑generated content (UGC) to manufacturable 3D models. It integrates requirement modeling, multimodal prompt engineering, 3D generation with DfM, and interpretable optimization.

About the Project

This repository accompanies the manuscript “Explainable Generative 3D Design Driven by Social Media Semantics” (under review). It provides runnable code, example data, and instructions to reproduce the main results. This website is a static companion page summarizing key information for reviewers.

  • R1: UGC‑driven requirement modeling with BERTopic and LLM structuring.
  • R2: Multimodal prompt engineering (text, image, hybrid).
  • R3: 3D generation with diffusion/NeRF and Design‑for‑Manufacturability checks.
  • R4: Interpretable optimization using Grad‑CAM and multi‑metric analytics.

Key Features

UGC → Semantic Specs

Transform noisy social media content into structured, traceable design requirements.

Multimodal Generation

Text‑to‑Image, Image‑to‑Image, and Text+Image fusion with diffusion priors and NeRF.

Explainability & DfM

Grad‑CAM overlays, quantitative metrics, and topology checks for manufacturability.

Quickstart

  1. Environment. Install Python ≥ 3.9. Optional: CUDA for GPU.
  2. Install. pip install -r requirements.txt
  3. Models. bash scripts/download_models.sh (instructions for any external weights).
  4. Sample data. Unzip examples/sample_data.zip to examples/sample_data/.
  5. Run demo. python tools/demo.py --mode ti2d --config configs/cox3d_demo.yaml
  6. Reproduce results. See docs/reproduction.md.

Datasets

We rely on public datasets (e.g., COCO, KITTI) and a minimal in‑house subset for case studies. Public datasets must be obtained from their official sources and used under their licenses. We provide minimal sample data in examples/sample_data.zip solely to verify the pipeline.
